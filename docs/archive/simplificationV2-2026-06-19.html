Executive Read

What is better now

  • AgentRuntime is much smaller than the original 21.7k-line starting point, with many tool families moved behind dedicated modules and parity tests.
  • Project-board planner prompts now share explicit prompt contract helpers instead of duplicating PM Review and planner card rules.
  • shared/types.ts is now a small compatibility barrel backed by domain-owned shared contract files.
  • registerIpc and buildSubagentReleaseGateReport now read as small assembly points backed by owned helper modules.
  • Validation discovery is script-backed through validation:inventory, validation:recommend, and validation:inventory:check.
  • Future agents can now identify likely local checks without scanning the whole package script list by hand.

What is still risky

  • Large renderer components still combine layout, state derivation, event wiring, and domain decisions in one declaration.
  • ProjectStore, registerMainIpc, and AgentRuntime.send remain central gravity wells.
  • Release evidence source-check maps are clearer but still dense enough to need domain-aware edits.
  • LLM agents will still be tempted to patch central files directly unless the next plan creates smaller ownership boundaries first.

Guiding Rules

Move first

Keep first slices behavior-preserving. Extract named units, preserve props and exports, then add focused cleanup only after parity tests are green.

Keep owners obvious

Put UI state in renderer-owned view models, IPC schemas beside their domain registrar, store mapping beside persistence helpers, and provider quirks in adapters.

Prefer narrow proof

Use validation:recommend to pick local checks, add pnpm run typecheck for TypeScript, and use Ambient with Kimi selected only when the affected path needs live Pi/provider behavior. GLM 5.1 is currently degraded; treat unavailable Ambient+Kimi as a provider blocker before any human-approved failover.

    • Do not change visual design, keyboard behavior, tooltip behavior, or top-level state ownership during the first pass.

Validation Strategy

Local first

  • Start each slice with pnpm run validation:recommend -- --domain <domain>.
  • Run the narrow test tied to the files touched, then pnpm run typecheck for TypeScript.
  • Use pnpm run complexity:inventory at phase boundaries, not after every tiny move.

Live only when relevant

  • For provider or Pi loop changes, use the real Ambient provider with Kimi selected. GLM 5.1 is currently degraded; if Ambient+Kimi is unavailable, record a provider blocker before any human-approved failover.
  • Do not expose API keys, credential file contents, shared snapshot paths beyond approved local references, or secret values in artifacts.
  • Do not create or modify .github/workflows for this plan.

Definition of Done

Reviewability

Top declarations shrink and future diffs mostly land in domain-owned files with obvious validation commands.

Behavior safety

Move-only extraction is backed by parity tests, UI-model tests, store/import-export tests, IPC tests, or runtime tests as appropriate.

LLM friendliness

An agent can modify one domain without loading a giant renderer component, central store, central IPC registrar, or full runtime loop into working memory.
