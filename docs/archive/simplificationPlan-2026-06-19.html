North Star

Desired outcome

Reduce the largest change surfaces into domain-owned modules so feature work lands in smaller, reviewable files with clear contracts and targeted tests.

Operating rule

Each phase starts as behavior-preserving extraction. Cleanups come only after parity tests show the moved code behaves exactly the same.

Diagram: Phased Roadmap

Sequence the work so every phase creates a safer base for the next one.

Implementation Phases

  1. Phase 0

    Install Safety Rails Before Moving Code

    Capture the current public surface and make future extractions mechanically checkable.

    • Add a lightweight IPC parity test that confirms `AmbientDesktopApi`, preload methods, and main-process channels stay aligned.
    • Add a script or Vitest helper that reports top file and declaration sizes, excluding generated files.
    • Document validation commands by domain, including Ambient provider checks with Kimi selected for provider-dependent validation.

    Exit gate: parity tests fail before behavior does.

  2. Phase 1

    Extract Renderer Workspaces from `App.tsx`

    Move large, already named UI units into domain files while preserving props, markup, state behavior, and CSS class names.

    • Start with `RightPanel`, then `AutomationsWorkspace`, then project-board panes.
    • Move pure presentational helpers next, leaving app-level state orchestration in `App.tsx` until stable.
    • Avoid redesign and state rewrites during this phase.

    2026-06-10 status: initial workspace extraction is complete enough to move on, but the extracted workspace files are now the renderer follow-up targets.

    Exit gate: renderer tests, UI-model smoke, and no visual surface changes except import paths.

  3. Phase 2

    Split Main IPC Registration by Domain

    Replace the single `registerIpc` growth point with domain registration functions that receive explicit dependencies.

    • Create `src/main/ipc/registerSettingsIpc.ts`, `registerProjectBoardIpc.ts`, `registerWorkflowIpc.ts`, and `registerBrowserIpc.ts` incrementally.
    • Keep `handleIpc` and trusted renderer enforcement centralized.
    • Move Zod schemas with their owning IPC domain, unless shared by multiple domains.

    2026-06-10 status: domain IPC modules exist for the main areas; remaining work is thinning large domain registrars and the central orchestration function.

    Exit gate: channel parity test passes and each moved domain has targeted main-process tests.

  4. Phase 3

    Thin `ProjectStore` into Persistence Facade plus Domain Helpers

    Keep `ProjectStore` as the public facade, but move pure mappers, migration SQL, and workflow-recording file logic to owned modules.

    • Extract project-board parse, normalize, and row-mapping helpers first.
    • Move workflow recording library helpers into `workflowRecordingLibrary.ts`.
    • Move schema and migration SQL into `projectStoreSchema.ts` after helper extraction stabilizes.

    2026-06-10 status: mapper and schema modules exist; keep future work facade-preserving and test-backed because ProjectStore remains a major hotspot.

    Exit gate: store tests and project-board import/export tests pass without public API changes.

  5. Phase 4

    • Begin with smaller extensions: provider catalog, media, voice, STT, and vision.
    • Keep new descriptor-based `registerTool` calls on the existing typed helper instead of reintroducing local casts.
    • Extract messaging tools only after the smaller modules prove the pattern.

Definition of Done

Reviewability

Top-level files shrink steadily, and most domain changes land in one or two obvious modules.

Safety

Move-only phases have parity tests, targeted unit tests, and live smoke tests only where the real loop is affected.

Ownership

Core semantics live in stable contracts, provider quirks live in adapters, and scenario-specific lessons live in tests or dogfood harnesses.

Prepared as a crisp phased implementation plan for low-risk codebase simplification. This document intentionally avoids secrets, live credentials, and cloud workflow changes.
